# Operating Workflow

- Start every non-trivial task with a clear contract: objective, scope in/out, affected services, validations, quality targets, and stop conditions.
- For cross-service, new integration, or high-risk changes, plan the architecture and review boundary impacts before implementation.
- Implement in small vertical slices and keep diffs focused.
- Run review and validation before marking work complete.
- Report changed files, verification results, and residual risks in every delivery summary.
