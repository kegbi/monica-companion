# Source of Truth

When implementing features or making architectural decisions, consult these documents in order:

1. `context/product/product-definition.md` — product scope, features, boundaries.
2. `context/product/roadmap.md` — implementation phases and priorities.
3. `context/product/architecture.md` — tech stack, infrastructure, and tooling decisions.
4. `context/product/service-architecture.md` — detailed service descriptions, responsibilities, communication flow, and caller allowlists.
5. `context/product/acceptance-criteria.md` — measurable "done when" criteria for v1.
6. `context/product/monica-api-scope.md` — Monica v4 API endpoints and contracts.
7. `.claude/rules/` — engineering rules and coding standards.
